Los tests E2E complementan esto verificando el **ciclo completo** contra un servidor real:
- Register → login → `/me` en una misma sesión HTTP con token transportado entre requests

**No hay duplicación**: los unit tests usan `TestClient` (FastAPI) y prueban lógica interna + monkeypatch de repos; los E2E usan Playwright `APIRequestContext` contra un servidor real.
